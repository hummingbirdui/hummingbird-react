## Default

`FormControl` renders a styled `<input>`.

<ComponentPreview name="form-control-default" className="max-w-sm mx-auto" />

## Variants

The `variant` prop switches between `outline` (default, bordered) and `fill` (a filled, muted surface).

<ComponentPreview name="form-control-variants" />

## Sizes

The `size` prop offers three sizes: `sm`, `md` (default), and `lg`.

<ComponentPreview name="form-control-sizes" />

## Colors

The `color` prop sets the focus accent: `primary`, `secondary`, `info`, `success`, or `warning`.

<ComponentPreview name="form-control-colors" />

## Validation state

The `state` prop marks a control `valid` or `invalid`, and pairs with a `FormText` of the matching `variant`. The `invalid` state also sets `aria-invalid`.

<ComponentPreview name="form-control-validation" />

## Textarea

`Textarea` shares the same styling for multi-line input.

<ComponentPreview name="form-control-textarea" className="max-w-sm mx-auto" />

## With label

`FormField` stacks a `FormLabel`, control, and `FormText`, and recolors the label to match the control on focus.

<ComponentPreview name="form-control-with-label" className="max-w-sm mx-auto" />

## With icon

`InputIcon` overlays an `InputIconStart` and/or `InputIconEnd` on a control and pads it to make room.

<ComponentPreview name="form-control-with-icon" />

### Textarea

Accepts the same `variant`, `size`, `color`, `state`, `asChild`, and `className` props as `FormControl`.

`FormField`, `FormLabel`, `InputIcon`, `InputIconStart`, and `InputIconEnd` are layout wrappers. Each accepts `asChild` and `className`.

## Styling

Hummingbird React form control is styled entirely through Hummingbird's utility classes and CSS
variables.

- See the full list of available form control classes in the [Class overview](https://hbui.dev/docs/forms/form-control/#class-overview).
- Visit the [CSS Variables](https://hbui.dev/docs/forms/form-control/#css-variables) documentation to explore all available variables.
