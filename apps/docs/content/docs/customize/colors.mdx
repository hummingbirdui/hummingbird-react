## Overview

Instead of applying fixed Tailwind palette classes like:

- `text-red-500`
- `bg-green-600`
- `border-blue-700` etc.

It is recommended to rely on semantic color classes. Semantic colors are mapped to design tokens and automatically adapt when themes change.

For example,

- `text-primary`
- `bg-subtle`
- `border-subtle` etc.

This approach prevents dependence on specific palette values (`blue-500`, `green-600`, etc.). Instead, components reference semantic roles (`primary`, `secondary`, `warning`), making the UI easier to maintain, brand, and theme.

Behind the scenes, each semantic color class points to a CSS variable. Updating or overriding that variable updates the color everywhere it is used — including inside every Hummingbird React component.

## How to use

Use semantic color classes like:

- `bg-primary`
- `text-muted`
- `border-default`

These classes use the theme's color variables, so the UI stays consistent when customizing or switching themes. They can be applied to any element or passed to any component through the `className` prop.

```tsx
<button className="bg-primary text-contrast hover:bg-primary-dark">
  Button
</button>

<p className="text-muted">Muted paragraph text</p>

<div className="border border-default bg-default p-4">
  Card with subtle border
</div>
```

## Customize color

Override any theme color or palette color by redefining its CSS variable inside a `@theme` block after importing Hummingbird:

```css
@import "tailwindcss";
@import "@hummingbirdui/hummingbird";
@source "../node_modules/@hummingbirdui/react";

@theme {
  --color-blue-500: oklch(65.2% 0.19 253.205);
  --color-purple-500: oklch(62.1% 0.253 305.351);

  --background-color-subtle: oklch(98% 0.02 240);
  --color-primary: oklch(65.2% 0.19 253.205);
  --color-secondary: oklch(62.1% 0.253 305.351);
}
```

## Color palettes

Each Hummingbird theme includes its own complete color palette, carefully designed to maintain a consistent visual tone. Use these palettes to customize existing themes or create entirely new ones. See the <a href="https://hbui.dev/docs/customize/colors#default-color-palette-reference" target="_blank" rel="noopener noreferrer">default color palette reference</a> in the Hummingbird documentation for the complete list of colors and their values.
